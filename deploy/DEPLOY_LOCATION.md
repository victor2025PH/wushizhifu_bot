# 部署目錄選擇說明

## 📁 為什麼推薦使用 /opt 目錄？

### /opt 目錄的優勢：

1. **Linux 標準規範**
   - `/opt` 是 Linux 文件系統層次標準（FHS）推薦的第三方軟件安裝目錄
   - 符合系統管理的最佳實踐

2. **組織結構清晰**
   - 系統文件、用戶文件、應用程序分離
   - 更易於管理和維護

3. **多用戶系統友好**
   - 不依賴特定用戶目錄
   - 適合生產環境

4. **權限管理**
   - 更靈活的權限設置
   - 適合系統服務

## 🏠 使用 /home/ubuntu 目錄的優勢：

1. **簡單直接**
   - 不需要 sudo 權限創建目錄
   - 操作更方便

2. **個人項目**
   - 適合測試環境
   - 適合單用戶使用

3. **備份簡單**
   - 用戶目錄通常一起備份

## 🎯 選擇建議

### 使用 /opt（推薦用於生產環境）：
- ✅ 生產服務器
- ✅ 多用戶系統
- ✅ 正式部署
- ✅ 需要系統級服務

### 使用 /home/ubuntu（適合開發測試）：
- ✅ 測試環境
- ✅ 個人項目
- ✅ 快速部署
- ✅ 單用戶使用

## 📋 兩個版本的部署腳本

我們提供了兩個版本：

1. **`deploy/full_deploy.sh`** - 部署到 `/opt/wushizhifu`
2. **`deploy/deploy_home.sh`** - 部署到 `/home/ubuntu/wushizhifu`

## 🚀 使用方式

### 部署到 /opt（生產環境）：
```bash
./deploy/full_deploy.sh
```

### 部署到 /home/ubuntu（開發測試）：
```bash
./deploy/deploy_home.sh
```

兩者的功能完全相同，只是項目目錄不同。您可以根據需求選擇。
